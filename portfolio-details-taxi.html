Project Overview

This end-to-end business intelligence solution optimizes taxi service operations through data-driven insights, predictive modeling, and automated KPI monitoring. The system processes NYC TLC taxi trip data to identify efficiency bottlenecks and service patterns, enabling better resource allocation and operational improvements.

Key Accomplishments

Advanced Predictive Models

Developed machine learning models using feature engineering and XGBoost that achieved 88.24% accuracy for demand prediction and 95.54% accuracy for revenue prediction. These models enhance resource planning capabilities by forecasting taxi demand patterns with high precision.

Anomaly Detection System

Created an automated anomaly detection system that identified 13 significant performance anomalies in the dataset that would have been missed with standard methods. This system allows stakeholders to quickly identify unusual patterns in taxi operations that could indicate problems or opportunities.

Real-time Monitoring & Recommendations

Developed an automated KPI tracking system with visualizations of critical performance metrics. The system generates actionable insights and data-driven recommendations for taxi fleet management and resource optimization, potentially improving fleet utilization by 25% during peak hours.

Technical Implementation

The project leverages advanced data processing and machine learning techniques to transform raw taxi data into actionable business intelligence:

Data Analysis Pipeline

The data pipeline processes NYC TLC taxi trip data to identify efficiency bottlenecks and service patterns. Key steps include:

  • Data cleaning and validation to handle outliers
  • Feature engineering with temporal patterns (hour of day, day of week)
  • Spatial analysis of pickup and dropoff locations
  • Statistical analysis to identify operational trends

Predictive Modeling

The predictive models forecast trip demand and revenue with high accuracy, enabling proactive resource planning:

  • XGBoost models with hyperparameter optimization
  • Feature importance analysis for model interpretability
  • Cross-validation for robust performance estimation
  • Time series forecasting for 7-day demand predictions

KPI Monitoring System

The monitoring system tracks critical metrics and alerts stakeholders about significant performance anomalies:

  • Real-time KPI tracking dashboard
  • Automated alerts for performance anomalies
  • Historical trend analysis with interactive visualizations
  • Customizable threshold settings for different metrics

Interactive Dashboard

The Streamlit dashboard visualizes performance metrics and provides data-driven resource optimization recommendations:

  • Interactive filters for different time periods and areas
  • Heatmaps showing demand hotspots across the city
  • Revenue and utilization metrics with trend analysis
  • Automated recommendations for optimal vehicle allocation

Running the Project

To run this project locally:

# 1. Clone the repository
git clone https://github.com/RohithIyer/taxi-service-optimization.git
cd taxi-service-optimization

# 2. Install requirements
pip install -r requirements.txt

# 3. Run the data processing
python src/data_processing/preprocess.py

# 4. Train the models
python src/modeling/demand_prediction.py

# 5. Run the KPI monitoring
python src/kpi_monitoring/monitor.py

# 6. Launch the dashboard
streamlit run dashboard/app.py

The project is modular in design, allowing you to run specific components independently. The Streamlit dashboard provides a comprehensive view of all analysis results and predictions.
